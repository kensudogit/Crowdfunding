# サンプルデータのセットアップガイド

## 問題: 画面にサンプル情報が表示されない

データベースにテストデータが登録されていない可能性があります。以下の方法で確認・解決できます。

## データベースの状態を確認

### 方法1: 確認スクリプトを使用（推奨）

**コマンドプロンプト（CMD）の場合:**
```bash
check-db.bat
```

**PowerShellの場合:**
```powershell
.\check-db.bat
```

このスクリプトで以下を確認できます：
- ユーザー数
- プロジェクト数
- 支援数
- コメント数

## サンプルデータを投入する方法

### 方法1: 管理画面から生成（最も簡単）

1. アプリケーションにログイン（任意のアカウントでOK）
2. ヘッダーの「管理画面」をクリック
3. 「サンプルデータを生成」ボタンをクリック
4. 生成完了メッセージを確認
5. プロジェクト一覧ページをリロード

### 方法2: サンプルデータ投入スクリプト

**コマンドプロンプト（CMD）の場合:**
```bash
init-sample-data.bat
```

**PowerShellの場合:**
```powershell
.\init-sample-data.bat
```

このスクリプトは：
- データベースコンテナが起動していることを確認
- `database/init/02_sample_data.sql` を実行してサンプルデータを投入

### 方法3: データベースをリセット（完全初期化）

既存データをすべて削除して、サンプルデータを投入します：

**コマンドプロンプト（CMD）の場合:**
```bash
reset-db.bat
```

**PowerShellの場合:**
```powershell
.\reset-db.bat
```

**注意**: この操作は既存のすべてのデータを削除します。

### 方法4: APIから直接生成

```bash
curl -X POST http://localhost:8000/api/sample-data/generate
```

または、ブラウザで以下にアクセス：
```
http://localhost:8000/api/sample-data/generate
```

## トラブルシューティング

### データベースコンテナが起動していない場合

**コマンドプロンプト（CMD）の場合:**
```bash
start-dev.bat
```

**PowerShellの場合:**
```powershell
.\start-dev.bat
```

### データが0件の場合

1. `check-db.bat` で確認
2. `init-sample-data.bat` でサンプルデータを投入
3. または管理画面から生成

### 重複エラーが発生する場合

既存データがある場合は、以下のいずれかを実行：
- `reset-db.bat` でデータベースをリセット
- 管理画面からサンプルデータを生成（既存データを削除してから生成）

## 確認方法

サンプルデータが正しく投入された場合：
- ユーザー数: 8人
- プロジェクト数: 10個
- 支援数: 50件以上
- コメント数: 複数

ブラウザで http://localhost:3000 にアクセスして、プロジェクト一覧が表示されることを確認してください。
